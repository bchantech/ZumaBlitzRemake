config/level_set.json

Contains information about level order and checkpoints.

A checkpoint is a place, where the player can start a new game.

  • The root object.
    • levelOrder: A list which contains level entries. Levels will be played in this order.
      • A single level entry. More info below.
    • checkpoints: A list of level order indices, where checkpoints should be placed.
        • If the player starts the game from a given checkpoint, they start from the entry the checkpoint is pointing to.
      • A single index corresponding to an entry in the levelOrder field.
    • startCheckpoints: A list of checkpoint indices, which are unlocked upon profile creation.
      • A single index, which corresponds to an entry in the checkpoints field.

Possible level order entry types

Debug information: this can be checked in Profile:getLevelName(), Profile:incrementLevel(), Profile:generateLevelID() and a number of other functions in Profile class.

level

A single level to be played.

  • A single level entry.
    • type: "level"
    • level: A level ID to be loaded and played.
    • name: A level name.
    • unlockCheckpointsOnBeat: If specified, after beating this entry, checkpoints with IDs corresponding to numbers in this list will be unlocked.
      • A single checkpoint to be unlocked after this entry has been completed.

randomizer

A group of levels to be played in random order.

  • A single level entry.
    • type: "randomizer"
    • pool: A list of level IDs to be chosen from.
      • A single level ID.
    • names: A list of level names. It is not shuffled, and this list should have the same length as the count field.
      • A single level name.
    • count: How many levels are to be picked from the pool.
    • mode: How the levels will be randomized. Available values are:
      1. "repeat" - Every next level is simply drawn at random from the list.
      2. "noRepeat" - Same as above, but an entry is removed from the pool once it's been picked. Requires the pool size to be greater or equal to the count field.
      3. "order" - Chooses entries from the pool at random without changing their order. Requires the pool size to be greater or equal to the count field.
    • unlockCheckpointsOnBeat: If specified, after beating this entry, checkpoints with IDs corresponding to numbers in this list will be unlocked.
      • A single checkpoint to be unlocked after this entry has been completed.
