config/shooters/*.json

Defines a Shooter.

  • The root object.
    • movement: Describes the positioning and movement of the shooter. More info below.
    • sprite: The sprite for the shooter to be used.
    • spriteOffset: The offset of the shooter sprite.
    • spriteAnchor: The anchor point of the shooter sprite.
    • shadowSprite: The shadow sprite for the shooter to be used.
    • shadowSpriteOffset: The offset of the shooter shadow sprite.
    • shadowSpriteAnchor: The anchor point of the shooter shadow sprite.
    • ballPos: The position of the ball which can be shot.
    • nextBallSprites: Contains next ball sprites for all possible sphere colors. Key names must be stringified integers.
      • sprite: A sprite used to draw the shooter's next color indicator.
      • spriteAnimationSpeed: When defined, the next sphere sprite will animate with that speed in frames per second.
    • nextBallOffset: The offset of the shooter's next ball sprite.
    • nextBallAnchor: The anchor point of the shooter's next ball sprite.
    • reticle: Describes the appearance of a shooting reticle.
      • sprite: The aiming reticle sprite to use when Aiming Retical is on. Will fallback to the default drawn caret if not defined.
      • offset: The aiming reticle's offset, relative to the position the cursor is aiming at.
      • nextBallSprite: The aiming reticle's next ball sprite to use when Aiming Retical is on. Will only be drawn if reticleSprite is defined.
      • nextBallOffset: The aiming reticle's offset, relative to the top-left of the aiming retical.
      • radiusSprite: The aiming reticle's sprite for spheres with ranged effects, like Fireballs or Color Clouds.
    • speedShotBeam: Describes the appearance of a speed shot beam.
      • sprite: The sprite to be used by the Speed Shot beam.
      • fadeTime: How long the speed shot beam should fade in or out, in seconds.
      • renderingType: Defines how the speed shot beam is rendered. Available values are:
        1. "full" - The full beam is always visible.
        2. "cut" - The beam is cut where it intersects a sphere train.
        3. "scale" - The beam is shrunk so its top is touching a sphere train.
      • colored: Whether the beam should be colored with the current sphere color.
    • sounds: Defines the sounds the shooter makes.
      • sphereSwap: A sound event played when the shooter's spheres are swapped.
      • sphereFill: A sound event played when the shooter is filled with newly generated spheres.
    • speedShotParticle: Persistent particle packet. If defined, it will spawn when a speed shot powerup is obtained, and despawn when the speed shot beam diminishes.
    • shotCooldown: The minimum time between the shots, in seconds. If multishot is set to false, this time will be counted since the previous shot sphere has been appended/disappeared.
    • shotCooldownFade: The duration of the ball appearing animation, in seconds. During this time you can't shoot balls yet. This time is counted after shotCooldown, so the effective cooldown is this plus that value.
    • multishot: If set, the player will be able to shoot another sphere before the previous one has hit the sphere train.
    • shootSpeed: The default speed of how fast the shot spheres will be, in pixels per second.
    • hitboxOffset: The offset of the Shooter's hitbox in pixels.
    • hitboxSize: The size of the Shooter's hitbox in pixels.

Possible shooter movement types

Debug information: this can be checked in Shooter:update(), Shooter:getInitialPos() and Shooter:getInitialAngle().

linear

The shooter will move on a line along the X axis.

  • A shooter movement.
    • type: "linear"
    • xMin: The minimum X position of the shooter. The player cannot move it past this value.
    • xMax: The maximum X position of the shooter. The player cannot move it past this value.
    • y: The Y position of the shooter.
    • angle: The rotation angle of the shooter, in degrees. 0 is up.

circular

The shooter will be stuck in one position and can freely rotate around the center point.

  • A shooter movement.
    • type: "circular"
    • x: The X position of the shooter.
    • y: The Y position of the shooter.
